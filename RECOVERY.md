# RECOVERY.md

## Purpose

Use this after a crash, disappearance, restart, gateway outage, or any moment where work may have continued without a proper user-facing wrap-up.

## Recovery Checklist

1. Read the recent chat context.
2. Inspect recent exec completions, subagent completions, and session events relevant to the active task.
3. Determine, concretely, which actions:
   - finished successfully
   - failed
   - are still running
   - never actually started
4. Send Richard a concise catch-up immediately.

## Mandatory Catch-Up Format

For each relevant task, report:
- what was attempted
- current status
- evidence (exit code, artifact, changed file, key result)
- next action, if any

## Rule

If there is any doubt whether Richard received a completion update, assume he did not and send one.
